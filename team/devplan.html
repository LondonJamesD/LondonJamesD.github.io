VSLR Development Plan

Naturoll

Product Overview

The VSLR product, 'Naturoll', will be a web-based card game where users collect and trade cards, partake in activities, and gain random cards.

MVP Deliverables

The first iteration (April 3rd) will feature a modern browser interface covering the core gameplay loop.

Frontend

Naturoll will feature a modern browser interface with a responsive design.

Backend

A thorough backend will be constructed to ensure smooth and safe delivery of the site.

Database

Clean, well-organized databases will make the product run clean and simple.

Core Loop

Includes acquiring random cards, browsing owned cards, personalized interfaces, and conducting trades.

System Architecture

A breakdown of the core services maintaining the platform ecosystem.

Account Management

Responsible for user authentication and profiles. Links inventory references with user accounts via a User ID.

Rewards Management

Responsible for user engagement, such as card pulls.

Inventory Management

Handles data regarding card access and ownership. Focuses on a simple frontend display and straightforward API integration.

Trade Management

Handles user trades and asynchronous execution. Uses transactions to ensure reliable, safe confirmation of trades.

The client requires a game that is simple enough yet has enough depth to justify continued playing, fitting our vision of building strategies via trading. The client wanted the game to be simple, fun, and have its own sort of user economy.

Revisions & Adjustments

Over the course of the development process, we had to revise our plan based on client feedback and timeline constraints.

Simplified Systems

Scrapped complex trading and reward mechanics in favor of a streamlined, simpler approach.

UI & Aesthetics

In response to client feedback, we refined the UI by adding different backgrounds and lighting in parts of the site.

Backend Optimization

Implemented a new search feature and an optimized weighted random selection SQL process for the card pull system to our client's request.

Balance & Limits

Reduced the overall scope of the MVP and added a strict time restriction on card pulls (2 minutes) to prevent server overload.
